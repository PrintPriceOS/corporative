# Hierarchy Amplification Rules

## General Principle
If the visual system is minimal (Monolith Pure), hierarchy must be achieved through structural decisions, not decorative additions.

## Rules
- **Spacing > Elements**: Increase negative space (padding/margins) to separate concepts before considering a divider or border.
- **Size > Color**: Increase font size or weight (within scale) to emphasize importance before adding accent colors.
- **Position > Decoration**: Use layout positioning (alignment/indentation) to create relationships before adding decorative symbols or icons.
- **Grouping**: Group related content into strict blocks using backgrounds (`surface_low` or `surface_dark`) to define functional zones.
- **Scanability**: Ensure the most critical technical label is the first thing eyes land on within a block.

## Process
1.  **Identify Importance**.
2.  **Apply Spacing**.
3.  **Apply Scale**.
4.  **Verify Contrast**.
5.  **Stop (Do NOT decorate)**.

---
*Never solve hierarchy with decoration. Spacing is the primary tool of the Monolith.*
