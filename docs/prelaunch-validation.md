# Pre-Launch Validation

## Goal
Validate if a first-time user can understand the platform nature and perform a primary action within 15 seconds.

## Test Users
- **Tier 1: Technical (CTO/Lead Dev)**: Focus on credibility, SDK visibility, and logic flow. 
- **Tier 2: Operational (Product Mgr/Ops)**: Focus on workflow automation and value-add.
- **Tier 3: Executive (Business/Decision Maker)**: Focus on high-level ROI and platform authority.

## Tasks

### Test 1 — Homepage Comprehension
**Ask**: "What is this platform and what is the first logical step you should take?"
**Expected**: 
- Identifies "Print Production Operating System" or similar.
- Selects "Start Production Audit" or "Explore Infrastructure".

### Test 2 — Action Intent
**Ask**: "Where would you go to scale your facility's budgeting?"
**Expected**: 
- Selects the Budget product card or clicks a CTA in the Budget section.

### Test 3 — Technical Activation (Developers)
**Ask**: "How long does it take you to find the first line of code to integrate?"
**Expected**: 
- Finds the Quickstart section in <5 seconds.

## Metrics
- **Comprehension Time**: Target <10s.
- **Hesitation Rate**: Number of pauses >3s at decision nodes.
- **Wrong Pathing**: Clicks that lead to unexpected secondary pages.
- **Visual Friction**: Parts of the page that are "too technical" to understand instantly.
